Getting the Best Results with Claude Code

üîÑ Reset Sessions Often

  • Start fresh between different tasks
  • Avoids unnecessary memory accumulation
  • Prevents context contamination
  • Keeps responses focused and accurate

üìù Write Descriptive Initial Prompts

  • Include context: what, where, why
  • Reference specific files and patterns
  • Share full error messages if debugging
  • Set clear boundaries and constraints

üéØ Use Planning Mode First

  • Always start with planning mode for new tickets
  • Review the plan carefully before coding
  • Refine iteratively until plan is solid
  • Better planning = better implementation

Planning mode breaks down tasks before writing code

Three rules for best results: Reset sessions between tasks to avoid memory bloat. Write detailed initial prompts with context and file paths. Always use planning mode first and refine the plan before coding.

How MCPs Work in Practice

Installation & Configuration

MCPs are configured in your Claude Desktop or Claude Code settings. Each MCP needs permissions and sometimes API keys.

Example: GitHub MCP

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_..."
      }
    }
  }
}

What You Can Do:

  • Read: "Show me issue #234"
  • Search: "Find files using Stripe API"
  • Create: "Make a PR for this fix"
  • Update: "Add comment to PR #45"
  • Analyze: "Review recent commits"
Pro tip: Start with read-only permissions, expand later
Configure MCPs in claude_desktop_config.json with command, args, and API tokens. Once set up, just ask Claude in plain English - it knows which MCP to use. Start with read-only permissions.

AI Agent Prompting Best Practices

üìù How to Prompt Effectively

  • Be specific: "Fix validation in UserController" not "fix the bug"
  • Set boundaries: "Only modify files in /src/auth/"
  • Point to files: "Check UserService.php line 234"
  • Share full error messages and stack traces
  • Reference patterns: "Follow OrderController example"

‚úÖ Review & Advanced Tips

  • Always test before accepting changes
  • Review diffs and understand all changes
  • Use CLAUDE.md for project conventions
  • Chain tasks: "Fix bug, add test, create PR"
  • Iterate - give feedback and let it improve

üéØ Claude Code Specific

  • Use Plan Mode: Let Claude think through the solution before coding
  • Enable thinking model: See Claude's reasoning process
  • Refine prompts iteratively: Start broad, then add specifics
  • Plan first, code later: Break down complex tasks in plan mode
Be specific with file paths and error messages. Use CLAUDE.md for conventions. For Claude Code: use plan mode first, enable thinking model, and always review changes before accepting.

MCP Security Best Practices

üîê Security Essentials

  • Start MCPs in read-only mode
  • Use separate API keys per service
  • Never share .env files with agents
  • Sanitize inputs - remove personal data
  • Grant minimal permissions needed

‚úÖ Operational Best Practices

  • Use pre-commit hooks to block secrets
  • Always review before approving changes
  • Test in dev/staging environments first
  • Track all MCP actions with git
  • Expand permissions gradually
Keep it simple: start with read-only mode, use separate API tokens, never share .env files, and always review changes. Pre-commit hooks prevent accidents. Expand permissions gradually as you build trust.

FINDINGS

Finding #1: The Fundamental Shift in Developer Work

The future: Less coding, more thinking and problem-solving

üîÑ The Traditional Developer Role

  • Heavy focus: Writing boilerplate code
  • Time spent: Typing implementations
  • Common tasks: Manual debugging & fixing
  • Limited: Planning and design time
  • Minimal: Code review attention

üöÄ The Emerging Developer Role

  • Heavy focus: Planning in plan mode
  • Time spent: Reviewing agent output
  • Common tasks: Refining prompts & iteration
  • Expanded: Architecture & design thinking
  • Critical: Quality assurance & review

Key insight: AI agents handle the typing, but developers must think deeper about problems, plan better, and review more carefully. The shift elevates the developer role from coder to architect and reviewer.

AI handles mechanical coding, developers focus on thinking and architecture. Plan mode forces better problem understanding upfront. The role shifts from "code writer" to "solution architect and quality guardian."

Finding #2: Understanding Matters MORE, Not Less

AI doesn't reduce the need for expertise - it amplifies it

üîç Why Code Review Is Critical

  • Agents can generate working code that violates your architecture patterns
  • Security vulnerabilities require human review (SQL injection, XSS, auth bypasses)
  • Performance implications aren't obvious to AI (N+1 queries, memory leaks)
  • Business logic edge cases need domain expertise
  • You're approving entire workflows, not just lines of code

üéØ Why Project Knowledge Enables Better Prompting

  • You know what to ask for: "Follow the pattern in OrderController"
  • You spot mistakes faster: "This breaks our caching strategy"
  • You set boundaries: "Only modify files in /src/features/payments"
  • You reference specifics: "Check UserService.php line 234"
The paradox: AI makes expertise MORE valuable. Seniors prompt better, catch issues faster, and understand implications. Code review is now about validating workflows and architecture, not syntax.

Finding #3: The Senior-Junior Prompting Gap

üë¥ Senior Developers

  • Know the codebase: Reference existing patterns
  • Set clear boundaries: "Only touch auth module"
  • Catch mistakes fast: Spot architectural issues
  • Iterate effectively: Give specific feedback
  • Use context: Point to files, functions, lines
Result: 3-5 iterations to working solution

üë∂ Junior Developers

  • Vague requests: "Fix the bug" or "Add validation"
  • Don't know patterns: Can't reference examples
  • Miss issues: Don't spot architectural problems
  • Generic feedback: "It doesn't work"
  • No context: Can't point to specific code
Result: 10+ iterations, or wrong solution accepted

üåâ Bridging the Gap

  • CLAUDE.md files: Codify senior knowledge & patterns
  • Pair prompting: Juniors watch seniors prompt
  • Prompt templates: "Always include file paths and line numbers"
  • Code review checklist: What to look for in agent output
  • Gradual autonomy: Start with simple tasks, expand slowly
  • Team retrospectives: Share what worked, what didn't
Seniors are 10x more effective because they know what to ask and spot issues fast. Bridge the gap with CLAUDE.md files, pair prompting, and templates. AI doesn't replace learning - it shifts what you need to learn.
